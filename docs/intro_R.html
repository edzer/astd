R, RStudio, R-Markdown

Always:

  • work from a script, don’t load/save workspace at startup/exit: Tools - Global options - Workspace:
    • deselect “Restore .RData into workspace at startup”
    • Save workspace to .RData on exit: choose never
  • work from an R-markdown (.Rmd or .qmd) file:
    • save this in a course-related directory
    • starting RStudio by clicking this file makes R use this as your working directory (default read/write of files)
    • NEVER use setwd() in an .Rmd file!
  • consider using quarto, https://quarto.org/

R objects

“Everything that is, is an object”

Assignment: these three statements are all equivalent:

a = 3
a <- 3
3 -> a

pipes

We use pipes for readable function composition, and can write

sin(sqrt(sum(3:7)))
[1] -0.9589243

alternatively as

3:7 |> 
  sum() |>
  sqrt() |>
  sin()
[1] -0.9589243

Assign-to-right may look better than to the left:

3:7 |> 
  sum() |>
  sqrt() |>
  sin() -> a
# versus:
a <- 3:7 |> 
  sum() |>
  sqrt() |>
  sin()

Recycling

  • all “scalar” objects are really vectors of length one. Almost all functions work on vectors.
  • recycle rule: in
a = 1:3
a + 1
[1] 2 3 4

the vector 1 is recycled to the length of a. There are catches:

a = 1:4
a + 1:2
[1] 2 4 4 6
a + 1:3
Warning in a + 1:3: longer object length is not a multiple of shorter object
length
[1] 2 4 6 5

Environments, Global Environment, scoping rules

By default, objects created are stored in the global environment, which is queried by

ls()
[1] "a"               "b"               "c"               "d"              
[5] "has_annotations" "l"               "ll"

R will look for objects all the way up in the search tree:

search()
 [1] ".GlobalEnv"        "package:sf"        "package:lubridate"
 [4] "package:forcats"   "package:stringr"   "package:dplyr"    
 [7] "package:purrr"     "package:readr"     "package:tidyr"    
[10] "package:tibble"    "package:ggplot2"   "package:tidyverse"
[13] "package:stats"     "package:graphics"  "package:grDevices"
[16] "package:utils"     "package:datasets"  "package:methods"  
[19] "Autoloads"         "package:base"

and this e.g. works:

a = 5
f = function(x) { x + a }
f(3)
[1] 8

although usually you don’t want this to work.

R functions

“Everything that happens, happens by a function call”

3 + 4
[1] 7

is equivalent to

`+`(3, 4)
[1] 7

and we can even redefine

`+`  = `-`
3 + 4
[1] -1

and remove this

rm(`+`)
